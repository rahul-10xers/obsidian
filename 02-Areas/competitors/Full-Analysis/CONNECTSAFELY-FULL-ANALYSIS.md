## 3. GEO STRATEGY (AI ENGINE OPTIMIZATION)

GEO = optimization for LLM answer engines (ChatGPT, Perplexity, Claude, Gemini). ConnectSafely is significantly ahead of most LinkedIn SaaS tools here.

### 3.1 Core GEO Signals

**Signal 1: Explicit llms.txt file**
URL: `https://connectsafely.ai/llms-full.txt`
A dedicated 99KB plain-text file containing the full product documentation in LLM-readable format. This is an explicit GEO play — they want AI engines to cite their product accurately.
Contents: complete feature documentation, statistics, pricing, tech stack, use cases, competitor framing.

**Signal 2: MCP Server offering**
URL: `https://connectsafely.ai/integrations/mcp-server`
Help center: `https://connectsafely.ai/help-center/integrations-api/mcp-server`
They offer a Model Context Protocol (MCP) server — meaning Claude, Cursor, and other AI coding tools can directly interact with their LinkedIn API. This positions ConnectSafely inside the AI development ecosystem.

**Signal 3: Claim density for AI citation**
From `https://connectsafely.ai/llms-full.txt`:
- "14.6% close rate (vs. 1.7% outbound)" — specific, citable, source-comparable
- "81% lower cost per lead ($35 vs $186)" — exact figures LLMs love to cite
- "zero bans across 1,000+ users" — categorical safety claim
- "1st position headline: '#1 LinkedIn inbound lead generation platform'" — self-declared category leader

**Signal 4: Structured product narrative for LLM retrieval**
The llms-full.txt uses clean markdown headers, bullet lists, and comparison tables — the exact format LLMs use for RAG retrieval. Every feature section starts with a "Purpose" line (LLM-optimized).

**Signal 5: Integration template pages (19 URLs)**
Pages like `/templates/n8n-linkedin-ai-connection-requests-gemini` — these pages use AI model names (Gemini) in URLs, which increases discovery via AI-related queries.

### 3.2 GEO Assessment

ConnectSafely is **deliberately building for AI answer engine visibility**. They have:
- ✅ llms.txt / llms-full.txt (explicit LLM-targeting)
- ✅ MCP server (embedded in AI tool ecosystem: Claude Desktop, ChatGPT, Cursor, VS Code)
- ✅ High claim density with specific numbers
- ✅ Integration with AI tools (Gemini, n8n AI workflows)
- ✅ **Schema.org structured data CONFIRMED** on multiple pages: `FAQPage`, `SoftwareApplication`, `BreadcrumbList`, `Organization`, `WebSite` schemas (source: `https://connectsafely.ai/comparison/best-expandi-alternative` schema data)
- ✅ **FAQ schema on all comparison pages** — 8 Q&A pairs per page, fully marked up for rich snippets
- ✅ **SoftwareApplication schema with pricing offers** — LowPrice $0, HighPrice $99, 4 offers, USD
- ❌ No Reddit/community GEO signal — zero Reddit presence found (confirmed via web search)
- ⚠️ **Stat inconsistency across LLM-facing content:** llms-full.txt states "350% average profile view increase" while homepage states "240% more profile views" — AI engines retrieving both will get conflicting data. This is a GEO weakness.
