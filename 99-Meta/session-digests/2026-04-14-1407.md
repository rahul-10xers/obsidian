# Session — 2026-04-14 14:07

## What we worked on
Built and debugged the LinkedIn Carousel Generator free-tools page (WP ID 1122) using the ACF programmatic pipeline. Uncovered and fixed a silent data-loss bug where WordPress was discarding all `ft_*` meta fields because they weren't registered for the REST API. Also fixed duplicate H1 and NeuronWriter placeholder tag issues in the content pipeline. Then pushed all ACF implementation docs to Mintlify and provided GitHub URLs for developer handoff.

## Key decisions made
- **Root cause of empty meta fields:** ACF `show_in_rest => 1` only covers `/wp-json/acf/v3/` — it does NOT register fields for the standard WP REST API at `/wp-json/wp/v2/`. WordPress silently discards unregistered meta keys on POST/PATCH with no error. Fix: `register_post_meta()` must be called for every `ft_*` field in an `add_action('init')` hook. This is now in `acf-field-group-snippet.php` (WPCode Snippet 1115).
- **H1 deduplication:** The PHP renderer already outputs `ft_hero_headline` in the `.ft-hero` section. The NeuronWriter HTML also contains an `<h1>` tag. `splitAtH2()` must strip the H1 from `preEmbedHtml` before storing — otherwise two H1s appear on page.
- **NeuronWriter `<imageplh>` tags:** NW embeds `<imageplh contenteditable="false"> </imageplh>` as image placeholders. These must be stripped from both pre and post embed HTML before pushing to WP — they render as broken markup outside the NW editor.
- **Mintlify nav is not auto-discovery:** Files in `docs/` are invisible to Mintlify unless explicitly listed under `navigation.groups[].pages[]` in `docs/docs.json`. Adding a file to the repo does not make it appear on the site.
- **qmd was never used in Apr 11–14 sessions** — vault knowledge was bypassed entirely. Technical learnings from this period were not captured as searchable vault notes.
